# A股选股器 设计文档

## 概述

每日盘前自动运行短线战法选股，将符合条件的A股筛选出来并通过网页展示，可分享给朋友使用。

## 技术栈

- **后端**: Python 3.11+ / FastAPI / SQLite (sqlalchemy)
- **数据源**: akshare（免费开源A股数据）
- **定时任务**: APScheduler
- **前端**: 纯 HTML + CSS + JavaScript（SPA，无框架依赖）
- **部署**: 单进程运行，可部署到 VPS / Docker / 云服务器

## 系统架构

```
数据层 (akshare) → 策略引擎 → FastAPI 服务 → 前端 SPA
                        ↕
                    SQLite 数据库
                        ↕
                  APScheduler (定时运行)
```

## 每日运行流程

### 第一批 - 盘前预选（7:00 - 7:30）
基于昨日完整日线数据运行的非竞价类战法。

### 第二批 - 竞价选股（9:25 - 9:28）
基于当日集合竞价数据运行的战法。

### 第三批 - 收盘复盘（15:30 - 16:00）
收盘后使用完整日线数据重新运行所有战法，同时统计今日表现：
- 今日预选股的实际涨幅排名
- 各策略今日胜率（入选后上涨 vs 下跌的比例）
- 今日最佳策略（选出数量 + 上涨比例综合评分）
- 炸板/坑人股复盘

### 数据获取方式
用户首次使用时，从 akshare 下载全市场股票历史 K 线数据（前 60 个交易日），缓存到本地 SQLite 数据库 `stock_cache` 表。每日增量更新最新数据。

### 存储估算与清理策略
- **日线缓存**：~5000只 × 60日 × 200B ≈ 60MB（首次填充）→ 后续每日增量极小
- **选股结果**：~1500条/日 × 300B ≈ 450KB/日 → 一年约 160MB
- **总计**：一年约 220MB，放10年不到 2GB，完全无需担心
- **自动清理策略**：
  - stock_cache 自动保留最近120个交易日数据
  - screening_results 保留最近365天，可配置
  - 管理员API可手动触发清理

## 15个战法详细定义

### 一、集合竞价类（需9:25后运行）

#### 1. 三一模式（集合竞价排名）
- **逻辑**: 集合竞价期间(9:15-9:25)，个股的成交额、换手率、涨幅三项指标排在全市场或同板块前沿
- **量化规则**:
  - 9:25竞价结束后获取竞价数据
  - 在全市场范围内，成交额排名前5%
  - 换手率排名前5%
  - 竞价涨幅排名前5%
  - 三项条件同时满足即为"三一"
  - 同板块内排名前三者加分

#### 2. 竞价爆量
- **逻辑**: 集合竞价成交量异常放大，资金抢筹信号
- **量化规则**:
  - 竞价量比 > 3.0
  - 竞价涨幅 > 2%（跳空高开）
  - 竞价成交量 > 昨日竞价成交量 × 2

#### 3. 竞价弱转强
- **逻辑**: 昨日炸板/走弱的个股，今日竞价从水下拉升至红盘，强弱逆转
- **量化规则**:
  - 昨日收盘涨幅 < 3% 或炸板（曾涨停但收盘未封住）
  - 今日竞价最低价 < -2%（水下）
  - 今日竞价最终价格 > +2%（拉红）
  - 竞价量比 > 2

### 二、连板接力类

#### 4. 一进二（首板抓二板）
- **逻辑**: 个股昨日首次涨停，今日有望走出连板
- **量化规则**:
  - 昨日首次涨停（非一字板，有换手）
  - 封板时间在14:00前
  - 昨日成交量温和放大（非天量）
  - 近10个交易日内未出现过涨停
  - 流通市值30-200亿
  - 同板块多只涨停则加分

#### 5. 二进三（二板抓三板）
- **逻辑**: 连续两个涨停后冲击三板，确认龙头地位的关键转折
- **量化规则**:
  - 昨日为二连板（连续2个涨停）
  - 换手率 > 10%（充分换手）
  - 非一字板（有真实成交）
  - 所属板块为当前热点板块
  - 在板块内辨识度高（板块龙头优先）

#### 6. 龙头战法
- **逻辑**: 识别热点板块中最强的龙头股
- **量化规则**:
  - 所属板块近期资金流入明显
  - 板块内有3只以上个股涨停
  - 该股在板块内涨幅最大、封板最早
  - 流通市值适中（50-200亿）
  - 连续涨停天数 > 跟风股

### 三、K线形态类

#### 7. 反包战法
- **逻辑**: 个股涨停失败/断板后，次日以强阳线反包前日高点
- **量化规则**:
  - 前日收阴线/炸板/断板（涨幅 < 3%或收阴）
  - 今日阳线收盘价 > 前日最高价
  - 今日成交量 > 前日成交量
  - 今日涨幅 > 3%

#### 8. 仙人指路
- **逻辑**: K线带长上影线，试盘信号，主力测试上方压力
- **量化规则**:
  - 上影线长度 > 实体长度 × 2
  - 上影线长度 > 振幅 × 60%
  - 股价处于底部或上涨初期（近60日价格区间的20%-50%分位）
  - 收盘价 > 开盘价（阳线）或跌幅 < 1%（小阴线也可以）
  - 成交量温和（非巨量）

#### 9. 多方炮（两阳夹一阴）
- **逻辑**: 两根阳线夹一根阴线，上涨中继形态
- **量化规则**:
  - 3日K线组合：阳→阴→阳
  - 第1日阳线涨幅 > 2%
  - 第2日阴线跌幅 < 3%（洗盘性质）
  - 第3日阳线收盘价 > 第1日收盘价
  - 第3日成交量 > 第2日成交量

#### 10. N字战法
- **逻辑**: 涨→回调→再突破前高，N字形走势
- **量化规则**:
  - 第一笔上涨：近10日内涨幅 > 10%
  - 回调：从最高点回调幅度 < 涨幅的60%（不破前低）
  - 第二笔上涨：今日放量突破前高
  - 今日成交量 > 20日均量 × 1.5

### 四、量价关系类

#### 11. 底部爆量
- **逻辑**: 底部长期横盘后突然放巨量阳线，主力建仓信号
- **量化规则**:
  - 股价处于近60日价格区间的下30%分位
  - 当日成交量 > 20日均量 × 2.5
  - 当日涨幅 > 2% 且收阳线
  - 当日成交量创近20日新高

#### 12. 量窒息（地量）
- **逻辑**: 成交量萎缩到极致，卖盘枯竭
- **量化规则**:
  - 当日成交量 < 20日均量 × 0.4
  - 成交量创近20日新低
  - 股价处于近30日低位（前期跌幅 > 10%）
  - 当日振幅 < 4%

#### 13. 放量突破
- **逻辑**: 关键阻力位配合放量突破，趋势确认
- **量化规则**:
  - 股价突破关键均线（MA20/MA60）或前高
  - 今日成交量 > 20日均量 × 1.5
  - 今日涨幅 > 3%

### 五、情绪/趋势类

#### 14. 龙回头
- **逻辑**: 前期龙头股大幅回调后企稳，有望走第二波
- **量化规则**:
  - 前期30日内最大涨幅 > 30%
  - 从最高点回调幅度 > 15% 且 < 40%（未破位）
  - 当前股价在MA20或MA60附近
  - 近3日成交量萎缩（< 20日均量 × 0.7）
  - 股价不再创新低

#### 15. 涨停基因（首板挖掘）
- **逻辑**: 近期有涨停记录的个股回调后可能再起
- **量化规则**:
  - 近20个交易日内出现过涨停
  - 涨停后最大回调幅度 < 15%
  - 当前股价在MA20上方
  - 今日成交量较前3日放大

## 前端设计

### 页面布局
- **顶部导航**: 应用名、当前日期、距离开盘倒计时、刷新按钮
- **日期切换**: 左右箭头切换日期，下拉查看所有有数据的日期
- **策略筛选栏**: 胶囊式按钮，"全部" + 各策略标签（附带数量）
- **股票卡片流**: 网格布局展示股票卡片
- **底部**: 数据来源、免责声明

### 股票卡片
- 股票名称 + 代码
- 当前价格 + 涨跌幅（红色涨/绿色跌）
- 触发策略标签（多个策略对应多个标签）
- 入选原因文字描述
- 关键辅助指标

### 其他视图
- 策略详情页（点击策略标签进入）
- 历史回顾（日期选择器）
- 个股详情（点击股票进入，追踪历史入选记录）
- 运行状态（今日筛选进度和结果统计）

## 部署与运维

### 运行方式
```bash
# 安装依赖
pip install -r requirements.txt
# 启动服务
python main.py
```

### 定时任务配置
- 服务启动后自动注册APScheduler任务
- 7:00 运行第一批选股（非竞价类：底部爆量、量窒息、一进二等）
- 9:25 运行第二批选股（竞价类：三一模式、竞价爆量、竞价弱转强）
- 提供手动触发API: POST /api/run-screening
- 错误重试：失败后10分钟自动重试，最多3次

### 部署建议
- 推荐部署到 VPS（最低配 1C2G 即可）
- 使用 systemd 或 Docker 管理进程
- 前端通过 Nginx 反向代理或直接 FastAPI 提供服务
