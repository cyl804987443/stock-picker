数据库表结构（SQLite）

screening_results（选股结果）
id | date | stock_code | stock_name | strategy_name | price | change_pct | volume_ratio
| reason | indicators_json | sector | market_cap | created_at
📌 每天每只股票每条策略一条记录，reason字段存入选原因，indicators_json存关键指标
daily_summary（每日统计）
id | date | total_stocks | strategies_count | run_at | status
stock_cache（日线缓存）
id | stock_code | trade_date | open | close | high | low | volume | amount
| ma5 | ma10 | ma20 | ma60 | macd | rsi | created_at
📌 缓存计算好的指标，避免重复请求akshare

REST API 接口

GET /api/results?date=2026-06-13&strategy=底部爆量
获取指定日期的选股结果，可按策略筛选
GET /api/strategies
获取所有策略列表及说明
GET /api/summary?date=2026-06-13
获取每日统计概览（总数、各策略数量）
POST /api/run-screening
手动触发选股（管理员用）
GET /api/dates
获取有选股记录的所有日期（用于日期导航）

数据库用 SQLite 足够了，一天几千条数据完全无压力。前端通过 5 个 API 就能搞定全部功能。
